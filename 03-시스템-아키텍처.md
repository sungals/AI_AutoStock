## 3. 데이터 수집 레이어 (Infrastructure)

```

  ├── DART API → 재무제표 (CFS 우선, OFS 폴백)

  ├── financial_statements 읽기
  ├── PER, PBR, ROE, 부채비율, 매출성장률 계산
  └── calculated_metrics 테이블 저장

[supply_demand.py]
  ├── pykrx → 투자자별 매매 (기관/외국인/개인)
  ├── pykrx → 공매도 현황
  └── investor_trading, short_selling 테이블 저장

[dart_disclosure.py]
  ├── DART API → 공시 목록
  └── dart_disclosures 테이블 저장

[macro_data.py]
  ├── yfinance → WTI, 금, 환율, BTC, VIX
  ├── FinanceDataReader → KS11, KQ11
  ├── FRED API → FFR, 실업률, CPI
  ├── ECOS API → 기준금리, CPI, M2
  ├── R-ONE API → 아파트 매매가격지수
  └── macro_prices 테이블 저장

[market_flow_collector.py]
  ├── pykrx → 시장 투자자별 매매 집계 (KOSPI/KOSDAQ)
  ├── pykrx → 프로그램 매매 (차익/비차익)
  ├── pykrx → 선물 OHLCV + 투자자별 선물 포지션
  └── market_investor_flow, program_trading, futures_data 저장

[news_crawler.py]
  ├── 네이버 검색 API → 뉴스 기사 (한도 내)
  ├── 크롤링 폴백 → 한도 초과 시
  ├── 네이버 토론방 크롤링 → 종목 토론
  └── news_articles, stock_discussions 저장
```

## 4. 분석 엔진 레이어 (Business)

```
[technical_analyzer.py]
  ├── price_data 읽기 (최근 500일)
  ├── ta 라이브러리 → 12종 기술지표 계산
  ├── 캔들스틱 패턴 4종 직접 구현
  └── technical_signals 테이블 저장

[sentiment_analyzer.py]
  ├── news_articles, stock_discussions 읽기
  ├── 한국어 긍정/부정/중립 키워드 사전 기반 분류
  └── sentiment_scores 테이블 저장 (composite_score)

[screening.py]
  ├── 11개 전략 함수 (screen_turnaround, screen_value, ...)
  ├── calculated_metrics + technical_signals + sentiment_scores 읽기
  ├── 각 전략 buy_score 0~100 계산
  └── screening_results 테이블 저장

[short_term_signal.py]
  ├── market_investor_flow 읽기 (자금흐름)
  ├── 단기 시그널 -100~+100 계산
  └── screening.py의 flow 전략 보너스 점수 제공

[flow_regime_analyzer.py]
  ├── market_investor_flow + macro_prices 읽기
  ├── 중장기 국면 (bull/bear/sideways) 판단
  └── 자동매매 엔진 및 포트폴리오 마법사에 제공

[empirical_analyzer.py]
  ├── simulation_runs + empirical_cases 읽기
  ├── 국면별 과거 전략 성과 분석
  └── 경험적 점수 계산 (역사적 성과 기반)

[fusion_analyzer.py]
  ├── technical_analyzer 점수 입력
  ├── empirical_analyzer 점수 입력
  ├── fusion_signals 테이블 저장
  └── 3-알고리즘 앙상블 → 최종 추천 및 신뢰도

[backtester.py]
  ├── price_data + screening_results 읽기
  ├── CAGR, MDD, Sharpe, Alpha 계산
  └── backtest_runs + backtest_metrics 저장

[simulation_runner.py]
  ├── ThreadPoolExecutor × 4 병렬 실행
  ├── walk-forward 방식으로 파라미터 공간 탐색
  └── simulation_runs + simulation_picks 저장

[guru_filter.py]
  ├── 피터 린치 PEG + 워런 버핏 질적 기준
  ├── lynch_score (0~100) + buffett_score (0~100) 계산
  └── calculated_metrics에 guru_score로 저장

[pairs_trading.py]
  ├── 동일 업종 종목 쌍 후보 생성
  ├── statsmodels coint() 공적분 검정
  ├── 반감기 + Z-score 시그널 계산
  └── pairs_trading_results 테이블 저장

[retail_defense.py]
  ├── 기관/외국인 매도 + 개인 매수 패턴 탐지
  ├── 개미 함정 리스크 점수 0~100 계산
  └── anti_trap_scores 테이블 저장

[signal_standardizer.py]
  ├── 11개 전략 모듈의 원시 점수 입력
  ├── -100 ~ +100 범위로 정규화 (Z-score 기반)
  └── signal_history 테이블 저장 (ML 학습용)

[flow_signal_advanced.py]
  ├── market_investor_flow + futures_data 읽기
  ├── 기관/외국인/개인 복합 자금흐름 시그널
  └── 선물 베이시스 + 풋콜비율 통합 분석
```

## 11. 핵심 설계 패턴

### 11.1 Connection Injection

DB 함수는 `conn` 파라미터를 외부에서 주입받아 트랜잭션을 호출자가 제어합니다.

```python
# 올바른 패턴
def save_screening_result(conn, corp_code: str, strategy: str, score: float):
    conn.execute("INSERT OR REPLACE INTO screening_results ...", (...))

# 호출 측에서 트랜잭션 제어
with db.get_connection() as conn:
    for result in results:
        save_screening_result(conn, ...)
    # 루프 완료 후 conn.__exit__에서 commit
```

### 11.2 CFS-first Fallback

재무제표는 연결재무제표(CFS) 우선, 없으면 개별재무제표(OFS) 조회.

```python
for fs_div in ('CFS', 'OFS'):
    data = fetch_from_dart(corp_code, year, fs_div)
    if data:
        return data
return None
```

### 11.4 Migration Pattern

`init_db()` 내 `_migrate_*()` 함수로 무중단 스키마 업그레이드.

```python
def _migrate_add_new_column(conn):
    cols = [r[1] for r in conn.execute("PRAGMA table_info(my_table)").fetchall()]
    if 'new_column' not in cols:
        conn.execute("ALTER TABLE my_table ADD COLUMN new_column TEXT")
```

### 11.6 Pure Functions (Risk Checks)

`portfolio_operations.py`의 리스크 체크 함수는 모두 순수 함수로 작성합니다.
사이드 이펙트 없이 입력만으로 판단하여 테스트가 쉽습니다.

```python
# 순수 함수 패턴 — DB 조회 없음, 부작용 없음
def check_trailing_stop(
    entry_price: float,
    highest_price: float,
    current_price: float,
    trail_pct: float = 0.08
) -> bool:
    """최고가 대비 trail_pct 이상 하락하면 True."""
    if highest_price <= 0:
        return False
    drawdown = (highest_price - current_price) / highest_price
    return drawdown >= trail_pct
```
