# DB 스키마

> SQLite 데이터베이스 전체 테이블 스키마입니다. 모든 CREATE TABLE 구문, 인덱스, 컬럼 타입을
> 포함합니다. `db_core.py`의 SCHEMA_SQL 및 마이그레이션 함수를 기준으로 작성되었습니다.

---

## 개요

| 항목 | 값 |
|------|-----|
| DB 엔진 | SQLite 3 |
| 저널 모드 | WAL (Write-Ahead Logging) |
| 파일 경로 | `{BASE_DIR}/quant_data.db` |
| 현재 크기 | ~756MB (KOSPI/KOSDAQ 10년치 데이터 기준) |
| 총 테이블 수 | 51 |

## 1. 기업 마스터 (companies)

KOSPI/KOSDAQ 전 종목의 기본 정보.

```sql
CREATE TABLE IF NOT EXISTS companies (
    corp_code   TEXT PRIMARY KEY,          -- DART 고유번호 (8자리)
    stock_code  TEXT NOT NULL UNIQUE,      -- KRX 종목코드 (6자리)
    corp_name   TEXT NOT NULL,             -- 기업명
    sector      TEXT,                      -- 업종
    industry    TEXT,                      -- 산업
    market      TEXT DEFAULT 'KOSPI',      -- 시장 구분 ('KOSPI' or 'KOSDAQ')
    updated_at  TEXT DEFAULT (datetime('now'))
);
```

## 2. 주가 데이터 (price_data)

일간 OHLCV + 시가총액 + 유통주식수.

```sql
CREATE TABLE IF NOT EXISTS price_data (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_code  TEXT NOT NULL,
    trade_date  TEXT NOT NULL,             -- 'YYYY-MM-DD'
    open        INTEGER,                   -- 시가 (원)
    high        INTEGER,                   -- 고가 (원)
    low         INTEGER,                   -- 저가 (원)
    close       INTEGER,                   -- 종가 (원)
    volume      INTEGER,                   -- 거래량
    market_cap  INTEGER,                   -- 시가총액 (원)
    shares_outstanding INTEGER,            -- 유통주식수
    updated_at  TEXT DEFAULT (datetime('now')),
    UNIQUE(stock_code, trade_date)
);

CREATE INDEX IF NOT EXISTS idx_price_stock_date ON price_data(stock_code, trade_date);
```

## 3. 재무제표 (financial_statements)

DART API에서 수집한 원본 재무제표 데이터. IFRS XBRL 구조를 그대로 저장합니다.

```sql
CREATE TABLE IF NOT EXISTS financial_statements (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    corp_code        TEXT NOT NULL,
    bsns_year        TEXT NOT NULL,        -- 사업연도 ('2025')
    reprt_code       TEXT NOT NULL,        -- 보고서 코드 ('11011'=사업보고서)
    fs_div           TEXT NOT NULL,        -- 재무제표 구분 ('CFS'=연결, 'OFS'=개별)
    sj_div           TEXT NOT NULL DEFAULT '', -- 재무제표 종류 ('BS', 'IS', 'CF' 등)
    account_id       TEXT NOT NULL,        -- IFRS 계정 ID
    account_nm       TEXT,                 -- 계정명 (한글)
    thstrm_amount    INTEGER,              -- 당기 금액 (원)
    frmtrm_amount    INTEGER,              -- 전기 금액 (원)
    bfefrmtrm_amount INTEGER,              -- 전전기 금액 (원)
    updated_at       TEXT DEFAULT (datetime('now')),
    UNIQUE(corp_code, bsns_year, reprt_code, fs_div, sj_div, account_id)
);

CREATE INDEX IF NOT EXISTS idx_fs_corp_year ON financial_statements(corp_code, bsns_year);
CREATE INDEX IF NOT EXISTS idx_fs_covering ON financial_statements(
    corp_code, bsns_year, reprt_code, account_id, thstrm_amount
);
```

## 4. 계산된 재무 지표 (calculated_metrics)

재무제표로부터 계산된 투자 지표들. 하나의 종목·날짜·지표당 1행.

```sql
CREATE TABLE IF NOT EXISTS calculated_metrics (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    corp_code    TEXT NOT NULL,
    stock_code   TEXT NOT NULL,
    calc_date    TEXT NOT NULL,            -- 계산 기준일 ('YYYY-MM-DD')
    metric_name  TEXT NOT NULL,            -- 지표명 ('per', 'pbr', 'roe', ...)
    metric_value REAL,                     -- 지표 값
    updated_at   TEXT DEFAULT (datetime('now')),
    UNIQUE(corp_code, calc_date, metric_name)
);

CREATE INDEX IF NOT EXISTS idx_metrics_corp_date ON calculated_metrics(corp_code, calc_date);
CREATE INDEX IF NOT EXISTS idx_metrics_stock ON calculated_metrics(stock_code, calc_date);
```

**주요 metric_name 값:**

| metric_name | 설명 |
|-------------|------|
| `per` | 주가수익비율 |
| `pbr` | 주가순자산비율 |
| `roe` | 자기자본이익률 (%) |
| `debt_ratio` | 부채비율 (%) |
| `opm` | 영업이익률 (%) |
| `revenue_growth_yoy` | 전년 대비 매출 성장률 (%) |
| `opincome_growth_yoy` | 전년 대비 영업이익 성장률 (%) |
| `momentum_1m` | 1개월 모멘텀 (%) |
| `momentum_3m` | 3개월 모멘텀 (%) |
| `sentiment_score` | 감성 점수 (-1.0 ~ 1.0) |
| `market_cap_krw` | 시가총액 (원) |
| `dividend_yield` | 배당수익률 (%) |
| `fcf` | 잉여현금흐름 |
| `quality_score` | 퀄리티 점수 |
| `volatility_20d` | 20일 변동성 (%) |
| `guru_score` | Guru Filter 종합 점수 |
| `lynch_score` | 피터 린치 점수 |
| `buffett_score` | 워런 버핏 점수 |
| `kelly_ratio` | Kelly Criterion 비율 |

```sql
CREATE TABLE IF NOT EXISTS screening_results (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    corp_code   TEXT NOT NULL,
    stock_code  TEXT NOT NULL,
    strategy    TEXT NOT NULL,             -- 전략명 ('turnaround', 'value', ...)

    signals     TEXT,                      -- 시그널 목록 (JSON 배열)
    screen_date TEXT NOT NULL,             -- 스크리닝 기준일
    updated_at  TEXT DEFAULT (datetime('now')),
    UNIQUE(corp_code, strategy, screen_date)
);

CREATE INDEX IF NOT EXISTS idx_screening_date ON screening_results(screen_date, strategy);
```

## 14. 시뮬레이션 실행 (simulation_runs)

Walk-forward 시뮬레이션 각 회차 결과.

```sql
CREATE TABLE IF NOT EXISTS simulation_runs (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    batch_id            TEXT,              -- 배치 실행 UUID
    strategy            TEXT NOT NULL,
    risk_profile        TEXT NOT NULL,     -- 'aggressive' | 'moderate' | 'defensive'
    window_months       INTEGER,           -- 백테스팅 윈도우 (개월)
    start_offset_months INTEGER,           -- 시작 오프셋
    tech_weight         REAL,              -- 기술 분석 가중치
    amplifier           REAL,              -- 동의 증폭 계수
    dampener            REAL,              -- 불일치 완화 계수
    cagr                REAL,              -- 연평균 복리 수익률 (%)
    mdd                 REAL,              -- 최대낙폭 (%)
    sharpe              REAL,              -- Sharpe Ratio
    alpha               REAL,              -- 대 KS11 Alpha
    n_trades            INTEGER,           -- 총 거래 횟수
    market_regime       TEXT,              -- 시뮬레이션 기간 국면
    status              TEXT DEFAULT 'ok', -- 'ok' | 'error'
    error_msg           TEXT,
    created_at          TEXT DEFAULT (datetime('now'))
);

CREATE INDEX IF NOT EXISTS idx_sim_runs_batch ON simulation_runs(batch_id);
CREATE INDEX IF NOT EXISTS idx_sim_runs_strategy ON simulation_runs(strategy, status);
```
