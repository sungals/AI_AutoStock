## cron 등록 + 라이브 풀런 (2026-06-19)

### cron
평일 16:10 KST(장마감 후) EOD 작업 등록 (`crontab`):
```cron
10 16 * * 1-5 cd .../app && .../app/venv/bin/python run_daily_pipeline.py --collect \
    >> .../logs/eod.log 2>&1 # TTAK_QUANT_EOD
```
- DB는 기본 `config.DB_PATH`(=`app/quant_data.db`). 첫 실행이 마스터 구축, 이후 증분 수집.
- ⚠️ macOS: `crontab` 수정·실행이 TCC(개인정보 보호)에 막힐 수 있음. 본 등록은 샌드박스
  해제로 설치 완료. 작업이 안 뜨면 시스템설정 > 개인정보 보호 > 전체 디스크 접근 권한에
  `/usr/sbin/cron` 추가.

### 라이브 풀런 (실제 KRX 데이터)
cron이 쓰는 동일 CLI로 대형주 20종목 × 2024-01~2026-06 수집·분석:
```
collect→screening→simulation→optimize→report 5단계 전부 completed
price_data: 11,980행 / 20종목 / 2024-01-02~2026-06-19
스크리닝: momentum 15 (value/turnaround=0 — 시총·DART재무 없음)
최우수(gate 통과): momentum/moderate OOS CAGR 200.6% / Sharpe 2.20 / DSR 1.000
```
**해석(중요)**: OOS CAGR 200%는 버그가 아니라 실데이터 산술 결과 — OOS 구간이 반도체
급등(SK하이닉스 142k→2,883k ≈ 20배, 삼성 79.6k→371k ≈ 4.6배)을 잡았다. IS는 −16%였다.
이는 신뢰성 레이어의 한계도 보여준다: overfitting gate는 노이즈 적합은 막지만, **20종목
집중·단일 강세 국면**의 결과를 '안전'하게 만들지는 못한다. 운영 시 생존편향 haircut·회전율·국면
인식이 추가로 필요(향후 과제).
- market_cap=0(KRX 시총 엔드포인트 장애 지속) → value/turnaround 미작동. DART 키 없어 재무 skip.
