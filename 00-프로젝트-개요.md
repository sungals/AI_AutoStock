# TTAK Quant — 프로젝트 개요

> 이 문서는 TTAK Quant 시스템을 처음부터 직접 구축하려는 개발자를 위한 배포 문서 패키지의 첫 번째 문서입니다.
> 소스 코드를 공유하지 않고 동일한 시스템을 독립적으로 구현하는 데 필요한 모든 개념을 담습니다.

---

## 1. 플랫폼 소개

TTAK Quant는 KOSPI / KOSDAQ 전 종목을 대상으로 하는 **한국 주식 퀀트 투자 플랫폼**입니다.
웹 브라우저에서 모든 기능에 접근할 수 있는 Flask 대시보드를 통해, 스크리닝부터 자동매매까지의
전체 퀀트 투자 워크플로우를 단일 시스템 안에서 제공합니다.

### 핵심 목표

| 목표 | 설명 |
|------|------|
| **데이터 통합** | DART 재무제표 + KRX 시세 + 매크로 + 뉴스를 단일 SQLite DB에 통합 |
| **멀티 전략 스크리닝** | 11종 전략을 매일 자동 실행하여 유망 종목 발굴 |
| **백테스팅 & 시뮬레이션** | 10,000회 walk-forward 시뮬레이션으로 전략 유효성 검증 |
| **포트폴리오 자동화** | 시장 국면 + 투자 성향 기반 자동 포트폴리오 구성 및 운영 |
| **자동매매 연동** | 한국투자증권 KIS OpenAPI 연동 (mock/live 전환 가능) |

## 2. 핵심 기능

### 2.1 스크리닝 엔진 (11개 전략)

매일 장마감 후 KOSPI/KOSDAQ 전 종목에 대해 아래 11개 전략을 자동 실행합니다.

| 전략명 | 핵심 로직 |
|--------|-----------|
| `turnaround` | 실적 전환주 — 영업이익 흑자전환, 매출 급증, ROE 개선 |
| `value` | 중장기 가치주 — 저PER + 저PBR + 고ROE + 저부채 |
| `momentum` | 단기 모멘텀 — 거래량 급증, 52주 신고가, 가격 돌파 |
| `sector` | 테마/업종 동반 상승 탐지 |
| `trend` | 추세 추종 — MA 골든크로스, MACD 시그널 |
| `mean_revert` | 평균 회귀 — RSI 과매도 반등, 볼린저 하단 이탈 |
| `event` | 이벤트 드리븐 — DART 공시, 실적 발표 |
| `volatility` | 변동성 돌파 — ATR 기반 변동성 확대 |
| `flow` | 수급 추세 — 기관/외국인 순매수, 단기 자금 흐름 |
| `quality_lowvol` | 퀄리티 & 저변동성 — 3개년 ROE/FCF 안정성 |
| `guru` | Guru Filter — 피터 린치 + 워런 버핏 수치화 |

각 전략은 `buy_score`(0~100)를 출력하며, 점수가 기준값 이상인 종목을 `screening_results` 테이블에 저장합니다.

### 2.2 3-알고리즘 융합 분석

단순 스크리닝 점수를 넘어, 세 가지 독립 알고리즘의 결과를 앙상블합니다.

```
기술적 분석 점수 (technical_analyzer)
        +
경험적 분석 점수 (empirical_analyzer)  →  융합 점수 + 신뢰도 + 추천
        +
융합 알고리즘 가중치 (fusion_analyzer)
```

- **동의 증폭**: 세 알고리즘이 일치할 때 신뢰도를 높여 시그널 강도 증폭
- **불일치 축소**: 알고리즘 간 방향이 엇갈릴 때 포지션 크기 자동 축소
- **시장 국면 연동**: bull/bear/sideways 국면에 따라 각 알고리즘의 가중치 동적 조정

### 2.3 Walk-Forward 10,000회 시뮬레이션

`ThreadPoolExecutor`를 활용한 병렬 시뮬레이션으로 전략별 실제 성과를 검증합니다.

- **파라미터 공간**: 전략 × 리스크 프로필 × 윈도우 기간 × 시작 오프셋
- **측정 지표**: CAGR, MDD, Sharpe Ratio, Alpha (대 KS11 벤치마크)
- **피드백 루프**: 시뮬레이션 결과로 전략 가중치를 자동 업데이트
- **경험 누적**: `empirical_cases` 테이블에 국면별 성과 기록, 다음 시뮬레이션에 활용

### 2.4 포트폴리오 관리

**Portfolio Wizard**: 시장 국면(bull/bear/sideways) + 투자 성향(공격/중립/방어)을 입력하면
최적 종목 배분을 자동 계산합니다.

- **멀티 타임프레임**: 장기(분기 리밸런싱) / 중단기(월 리밸런싱) / 단타(주 리밸런싱) 3분할
- **Kelly Criterion**: 승률과 손익비 기반 포지션 크기 자동 계산
- **운영 알고리즘**: Trailing Stop, Fundamental Stop, Timeout Exit, De-leveraging, VIX 방어 모드
- **Pairs Trading**: 공적분 기반 통계적 차익거래 (Z-score 시그널)

### 2.5 자동매매 (KIS OpenAPI)

한국투자증권 OpenAPI를 통해 실시간 시세 수신 및 매수/매도 주문을 자동화합니다.

- **Mock 모드**: 실제 주문 없이 전략 검증 (기본값, 안전)
- **Live 모드**: 실제 계좌 연동 (환경변수로 전환)
- **5단계 일일 실행**: 포지션 점검 → 손절 체크 → 리밸런싱 → 신규 매수 → 보고
- **리스크 가드**: 최대 투자 비율, 일일 손실 한도, 최대 포지션 크기 설정

### 2.6 시그널 표준화 엔진 (Phase A)

11개 전략 모듈 각각이 `-100 ~ +100` 범위의 정규화된 시그널 점수를 출력합니다.
이 점수들은 `signal_history` 테이블에 매일 저장되어 향후 ML 모델의 학습 데이터로 활용됩니다.

### 2.7 보조 분석 모듈

| 모듈 | 역할 |
|------|------|
| **Guru Filter** | 피터 린치 PEG + 워런 버핏 질적 기준으로 종목 필터링 |
| **개미방어 (Anti-Trap)** | 기관/외국인 매도 + 개인 순매수 패턴 탐지 → 함정 리스크 점수화 |
| **테마 키워드** | 뉴스 + Google/네이버 트렌드에서 테마 키워드 추출·점수화 |
| **알림 시스템** | 국면 전환, 목표가 도달, 파이프라인 완료 시 웹 알림 |
| **AI 일일 분석** | Claude API로 시장 국면 평가 + 전략 조합 추천 (선택) |

## 3. 데이터 수집 범위

### 종목 데이터

| 데이터 | 수집 방법 | 빈도 |
|--------|-----------|------|

## 4. 시스템 요구사항

### 최소 사양

| 항목 | 요구사항 |
|------|----------|
| **Python** | 3.9 이상 (3.9 기준 타입 힌트 작성: `Optional[X]`, `float\|None` 금지) |
| **디스크** | 1GB 이상 (SQLite DB ~756MB, 데이터 증가 감안) |
| **RAM** | 4GB 이상 (10,000회 병렬 시뮬레이션 시 ~2GB 사용) |
| **CPU** | 2코어 이상 (ThreadPoolExecutor × 4 워커) |
| **OS** | Linux (Ubuntu 22.04 권장), macOS 가능 |

### 권장 사양 (프로덕션)

| 항목 | 권장 사양 |
|------|-----------|
| **RAM** | 8GB |
| **저장소** | SSD 20GB (DB 증가분, 백업 포함) |
| **네트워크** | 안정적인 인터넷 연결 (DART API, yfinance, pykrx 수집) |

### 소프트웨어 스택

```
Python 3.9+
SQLite 3 (WAL 모드, WAL 저널링)
Nginx (리버스 프록시)
Gunicorn (WSGI 서버, workers=2)
Cron (일일 파이프라인 자동 실행)
```
