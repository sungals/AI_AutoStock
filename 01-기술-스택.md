## 3. 데이터베이스

### SQLite3 (표준 라이브러리)

추가 설치 없이 Python 내장 `sqlite3` 모듈을 사용합니다.
프로덕션에서도 SQLite로 충분합니다 (단일 서버, 동시성 낮음).

**핵심 PRAGMA 설정:**

```python
conn.execute("PRAGMA journal_mode=WAL")       # Write-Ahead Logging 활성화
conn.execute("PRAGMA busy_timeout=30000")     # 잠금 충돌 시 30초 대기
conn.execute("PRAGMA synchronous=NORMAL")     # WAL 모드에서 성능/안전 균형
conn.execute("PRAGMA cache_size=-64000")      # 64MB 페이지 캐시
conn.execute("PRAGMA foreign_keys=ON")        # 외래키 제약 활성화
conn.row_factory = sqlite3.Row                # 결과를 dict처럼 접근
```

**Connection Injection 패턴:**

모든 DB 함수는 `conn` 파라미터를 외부에서 주입받아 트랜잭션을 호출자가 제어합니다.

```python
# 올바른 패턴
def upsert_price(conn, stock_code: str, trade_date: str, close: int):
    conn.execute("INSERT OR REPLACE INTO price_data ...", (...))

# 사용 시
with get_connection() as conn:
    upsert_price(conn, '005930', '2026-03-05', 78000)
    upsert_price(conn, '000660', '2026-03-05', 195000)
    # conn.commit()은 get_connection() 컨텍스트 매니저가 처리
```

**마이그레이션 패턴:**

스키마 변경 시 `init_db()` 내에 `_migrate_*()` 함수를 추가합니다.
`ALTER TABLE ADD COLUMN`은 멱등적으로 작성하여 무중단 업그레이드를 보장합니다.

```python
def _migrate_add_market_column(conn):
    """Add market column to companies if missing."""
    cols = [r[1] for r in conn.execute("PRAGMA table_info(companies)").fetchall()]
    if 'market' not in cols:
        conn.execute("ALTER TABLE companies ADD COLUMN market TEXT DEFAULT 'KOSPI'")
```

### 실시간 업데이트 (SSE)

Server-Sent Events로 파이프라인 진행상황 및 KIS 실시간 시세를 전송합니다.

```python
# Flask SSE 엔드포인트 패턴
from flask import Response, stream_with_context

@app.route('/api/pipeline/stream')
def pipeline_stream():
    def generate():
        while True:
            event = get_next_event()
            yield f"data: {json.dumps(event)}\n\n"
            if event.get('done'):
                break
    return Response(stream_with_context(generate()),
                    mimetype='text/event-stream')
```

```javascript
// JavaScript SSE 클라이언트 패턴
const evtSource = new EventSource('/api/pipeline/stream');
evtSource.onmessage = (e) => {
    const data = JSON.parse(e.data);
    updateUI(data);
    if (data.done) evtSource.close();
};
```

### WSGI 서버

**Gunicorn** — 프로덕션 WSGI 서버

```bash
pip install gunicorn
```

권장 실행 설정:

```bash
gunicorn web_app:app \
  -b 127.0.0.1:5000 \
  --workers 2 \
  --timeout 60 \
  --daemon \
  --log-file /tmp/gunicorn.log
```

workers=2는 SQLite WAL 모드와 안전하게 동작합니다.
workers를 더 늘리면 SQLite busy_timeout이 빈번하게 발생할 수 있습니다.
